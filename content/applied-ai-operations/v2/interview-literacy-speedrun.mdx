# Interview Literacy Speedrun

The 48-hour core remains independently accessible. It is designed to make a zero-knowledge learner safer and more articulate in an interview, not to manufacture a claim of production engineering experience.

## 48-hour core

1. Role map and truthful technical-bridge narrative
2. Application map and Python code literacy
3. LLM behavior, prompts, structured outputs, and tradeoffs
4. Retrieval, citations, tools, and deterministic workflows
5. Sensitive-data boundaries, authorization, injection, and approvals
6. Evaluation, rollback, monitoring, and escalation
7. System design from discovery to controlled pilot
8. Project defense, behavioral pressure, and full mock loop

## Optional Days 3–7

- Guided walkthrough of the two fictional local prototypes
- Practical API reading, validation, tests, and tracebacks
- Retrieval and evaluation deepening
- Boundary drill where conventional automation or no AI can be correct
- Timed system-design, project-defense, and full mock practice

Sprint completion awards only **Know** and selected **Practice** evidence. It creates an interview packet, not a certification or independent implementation claim.
