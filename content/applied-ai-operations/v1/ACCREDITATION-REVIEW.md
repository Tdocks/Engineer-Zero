# Applied AI Operations Engineer — Accreditation Review

**Status:** remediation in progress — not eligible for commercial certification

## Release blockers

| ID | Finding | Required evidence before release |
| --- | --- | --- |
| AIO-01 | Baseline answers must not be available in the browser bundle. | Server-only answer keys, public attempt projection, and a passing leakage test. |
| AIO-02 | Required work cannot pass by repeating headings alone. | Activity-specific rules plus a saved revision trail. |
| AIO-03 | Lessons must be independently reviewable rather than generic cards. | Four instructional sections, worked example, misconceptions, source, three checks, and artifact rubric per module. |
| AIO-04 | Missions must model choices and consequences. | Three options per decision, progressive reveal, consequence display, and a conventional/no-AI valid path. |
| AIO-05 | Certification cannot be automatic. | Two case studies, capstone evidence, three interview responses, readiness threshold, and qualified reviewer approval. |

## Required human review

Every published module, lab, mission, and assessment form must be approved by:

1. an instructional designer for objective-to-assessment alignment;
2. an applied-AI SME for technical accuracy and role relevance;
3. an accessibility reviewer for keyboard, reading-order, contrast, and reduced-motion behavior;
4. a fictional-data reviewer for safe scenario boundaries.

## Learner-experience gate

The release candidate must pass desktop and mobile walkthroughs for onboarding, baseline assessment, Sprint, Academy, each lab mode, Mission, Projects, Interview Studio, Readiness, error/retry, reset, and resume. Any answer-position cue, inaccessible control, missing recovery path, or unclear completion state is a release blocker.
